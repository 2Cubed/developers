Class: Channel

Channel

new Channel(id, options)

The channel class is responsible for maintaining the state of a channel's chat. This includes saying a listing of users, listening for events that are sent over redis, and propogating those events to users.
Parameters:
Name Type Description
id Number The channel Id
options Object Extra options for the channel.

Methods

broadcast(event, data)

Broadcasts an event to all users connected to the current chat server. This does *not* emit an event to every user in the chat. For that, see Channel.emit.
Parameters:
Name Type Description
event String
data *

Type

emit(event, data)

Emits an event to Redis for this channel. It can be listened for in widgets - see the example. Using this method is the recommended way of broadcasting events across a channel's chat servers.
Parameters:
Name Type Description
event String The event name.
data Object The data.
Example
channel.emit('FooBar', { foo: 'bar' });
// in a widget...
hook.event('FooBar', function (channel, data) {
    console.log(data); // => { foo: 'bar' }
});

forUser(fn)

Iterates fn through every client connected to the channel. Note that if a user is connected in multiple browser windows or locations, each session will be iterated through separately.

getRedis() → {external:"Redis"}

Returns the node-redis client associated with the channel. Note that the client has been promisified by Bluebird, so you can call things like getAsync(key), which return a promise.
Returns:
Type
external:"Redis"

getUsers() → {Array}

Returns an object of users. The keys will be in the format "u" and the values will be an array containing User objects, one for each session that user ID is in. Note that every array may not contain even one user. We do cleanups periodically, but until a cleanup is run, when a user leaves a chat their array will be empty. Also, this only contains users connected to the _current_ chat server.
Returns:
Type
Array

sendChatMessage(user, msg)

Sends a chat message from the user's account. You can pass in a user object, or any object that has properties "username", "id", and "role".
Parameters:
Name Type Description
user Object The user
msg String The message.
