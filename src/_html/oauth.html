Introduction

When you need access to a users' private resources, need to be able to act as another user in chat or just want to use Beam credentials to authenticate your users, it's best to use OAuth so the users' credentials don't get stored on your servers.

OAuth is a system that replaces the credentials by app-specific tokens. Your app can request a set of tokens for a certain user with a specific set of permissions. This way your app only gets access to what it needs and all other details will stay hidden.

To start using OAuth you should start by creating an application as described below.

Registering your application

To create an application you can go to the page called Manage OAuth. (You might need to authorize the developers site first). Once on the page click the Create New button in the top right corner and it'll open the creation form.

On this page you'll need to enter some basic details about your application, like the name, website and logo for your application. All of these will be displayed to the user and are for this reason required.

Hosts

The hosts parameter is an important one. This tells us what domains your application can redirect to. (We'll explain redirection later) This should be domains you control. You can use wildcards, for example: *.beam.pro will allow redirects to all subdomains on beam.pro, but not beam.pro itself. The list of domains is comma separated if you want to add multiple ones.
